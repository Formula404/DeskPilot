# DeskPilot 项目骨架设计

## 1. 目标

本文档定义第一版项目骨架。骨架目标不是完成全部功能，而是打通最小闭环：

```text
Tauri 悬浮窗
  -> FastAPI 本地服务
  -> LangGraph tool calling Agent
  -> LLM 调用浏览器扩展通道采集当前网页
  -> LLM 调用文件工具写入 Markdown
  -> SSE 回传状态
```

## 3. 后端启动边界

后端入口固定为：

```text
backend/app/main.py
```

本地服务默认监听：

```text
127.0.0.1:8765
```

Tauri 和浏览器扩展只允许访问本地服务，不直接访问数据库。

Tauri 使用 HTTP + SSE；浏览器扩展使用 WebSocket 作为主通道，HTTP 兼容接口只用于开发期调试和兜底。

## 4. 第一版模块依赖方向

```text
api -> agent -> tools -> context/rpa/db
api -> safety
agent -> safety
agent -> memory
tools -> safety
tools -> db
```

禁止反向依赖：

- `tools` 不能依赖 `api`。
- `context` 不能依赖 `agent`。
- `rpa` 不能依赖 `agent`。
- `frontend` 不能访问数据库。
- `browser-extension` 不能访问数据库。

## 5. 第一版核心流程

```text
POST /chat
  -> 创建 task_run
  -> 获取当前上下文
  -> LangGraph 选择 intent 和工具白名单
  -> OpenAI tool calling loop
  -> Tool Registry 执行工具
  -> 写 task_steps
  -> SSE 推送事件
  -> 生成最终结果
```

## 6. 第一批工具

必须先实现：

- `desktop.get_foreground_window`
- `browser.collect_current_page`
- `browser.get_current_page`
- `browser.summarize_current_page`（兼容/兜底）
- `browser.execute_page_action`
- `file.write_markdown`
- `file.write_xlsx`

第一条功能链路中，`web_page_summary` 只向 LLM 暴露：

- `browser.collect_current_page`
- `file.write_markdown`

第二批实现：

- `browser.extract_tables`
- `browser.export_table_to_xlsx`
- `desktop.open_app`
- `music.netease.play_song`

第三批实现：

- `wechat.export_visible_messages`
- `rpa.click`
- `rpa.type_text`
- `rpa.ocr_region`
