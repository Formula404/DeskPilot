# DeskPilot 开发规范

## 1. 目标

本文档定义 DeskPilot 第一版开发时必须遵守的工程规范。技术选型以 `report/技术方案.md` 为准，本文只约束代码组织、命名、接口、配置、日志、测试和安全边界。

## 2. 第一版固定技术栈

- 桌面端：Tauri + React + Vite + TypeScript。
- 前端状态：Zustand。
- 前端样式：Tailwind CSS + shadcn/ui。
- 后端：Python 3.12 + FastAPI。
- Python 包管理：uv。
- Agent：LangGraph。
- 模型：OpenAI SDK。
- 数据库：SQLite。
- 长期记忆检索：SQLite FTS5。
- 浏览器通道：Chrome/Edge Manifest V3 扩展 + 本地 WebSocket/HTTP。
- 浏览器自动化：扩展执行用户当前页轻量动作 + Playwright 受控浏览器。
- Windows 自动化：pywin32 + pywinauto。
- RPA 视觉：mss + opencv-python + PaddleOCR。
- 鼠标键盘：pyautogui。
- 表格导出：openpyxl。
- 实时事件：SSE。
- 测试：pytest。

## 3. 仓库目录规范

第一版目录固定如下：

```text
DeskPilot/
  backend/
    app/
      api/
      agent/
      context/
      core/
      db/
      memory/
      safety/
      tools/
      rpa/
      schemas/
    tests/
  frontend/
    src/
      api/
      components/
      features/
      store/
      styles/
      types/
  browser-extension/
    src/
    public/
  data/
    config/
    exports/
    logs/
    screenshots/
    tasks/
  report/
```

约束：

- `backend/app/agent/` 只放 LangGraph 图、节点、状态定义和意图路由。
- `backend/app/tools/` 只放可被 Agent 调用的结构化工具。
- `backend/app/rpa/` 只放底层截图、OCR、模板匹配、点击、输入等动作能力。
- `backend/app/context/` 只放当前窗口、浏览器上下文、截图上下文采集能力。
- `backend/app/safety/` 只放权限判断、审批、风险等级和敏感操作拦截。
- `frontend/` 不直接调用 OpenAI，不直接执行本机自动化。
- `browser-extension/` 负责浏览器侧通信、当前网页上下文读取和被授权的页面内动作，不实现 Agent 逻辑。

## 4. Python 代码规范

- 使用类型标注。
- 数据结构使用 Pydantic model。
- Agent state 使用 `TypedDict` 或 Pydantic model，保持字段稳定。
- 工具输入输出必须是结构化 schema，禁止让 LLM 直接拼接 shell 命令。
- 新增工具必须注册到 Tool Registry。
- 新增工具必须声明权限等级、风险等级、可观察结果和失败处理方式。
- 文件路径使用 `pathlib.Path`。
- 日志使用 Python logging。
- 禁止在工具中静默吞异常，必须返回结构化错误。

## 5. TypeScript 代码规范

- 使用 TypeScript strict mode。
- API 类型集中放在 `frontend/src/types/`。
- 后端请求集中放在 `frontend/src/api/`。
- Zustand store 只保存 UI 状态、任务状态和设置，不保存大段敏感内容。
- 组件按功能拆到 `frontend/src/features/`。
- 通用 UI 放 `frontend/src/components/`。
- 不在组件里直接拼 URL，统一走 API client。

## 6. Agent 开发规范

Agent 采用“意图识别 + LangGraph workflow 主干 + 受限 ReAct 子循环”。

约束：

- 任务必须先进入意图路由。
- 已知任务必须优先走固定 workflow。
- ReAct 只允许在 workflow 的局部节点里使用。
- ReAct 最大步数第一版固定为 5 步。
- 每次工具调用前必须经过权限检查。
- 每次工具调用后必须产生 observation。
- 敏感任务必须进入 approval 节点。
- Agent 不允许直接执行任意 Python、PowerShell、cmd 或 shell 字符串。

第一版 intent 分类：

- `web_page_summary`
- `web_table_export`
- `web_page_action`
- `desktop_app_open`
- `music_play`
- `wechat_visible_export`
- `file_operation`
- `general_chat`
- `unknown`

## 7. 工具开发规范

每个工具必须包含：

- `name`
- `description`
- `input_schema`
- `output_schema`
- `risk_level`
- `required_permissions`
- `handler`
- `observe`

工具命名格式：

```text
domain.action
```

示例：

```text
browser.get_current_page
browser.summarize_current_page
browser.export_table_to_xlsx
desktop.get_foreground_window
music.netease.play_song
wechat.export_visible_messages
file.write_markdown
```

工具返回格式：

```json
{
  "ok": true,
  "data": {},
  "message": "",
  "artifacts": [],
  "error": null
}
```

失败时：

```json
{
  "ok": false,
  "data": null,
  "message": "human readable reason",
  "artifacts": [],
  "error": {
    "code": "ERROR_CODE",
    "detail": {}
  }
}
```

## 8. RPA 开发规范

RPA 执行链路固定为：

```text
截图
  -> 预处理
  -> OCR / 模板匹配 / UIA 查询
  -> 定位目标
  -> 执行动作
  -> 再截图
  -> 校验结果
```

约束：

- 优先使用 UIA，其次 OCR/模板匹配，最后才使用坐标。
- 坐标必须使用窗口相对坐标。
- 禁止写死全屏绝对坐标。
- 每个动作必须有超时。
- 每个动作必须支持取消。
- 每个 RPA workflow 必须有最大步数。
- 敏感操作执行前必须请求用户确认。
- RPA 截图默认只保存到任务目录，不能写入长期记忆。

## 9. 浏览器扩展规范

浏览器扩展第一版支持 Chrome/Edge Manifest V3。扩展是浏览器侧 bridge，不是用户任务入口；用户只在 DeskPilot 悬浮窗/应用点击发送，后端通过扩展通道请求采集或动作执行。

职责：

- 建立与本地 FastAPI 的 WebSocket 通道，HTTP 仅作兼容/兜底。
- 根据后端结构化指令获取当前 tab 的 URL、标题、可见文本、DOM 摘要。
- 根据后端结构化指令执行被授权的轻量页面内动作。
- 将采集结果、动作结果和错误回传给本地服务。

约束：

- 扩展不调用 OpenAI。
- 扩展不保存长期数据。
- 扩展不自行规划任务。
- 扩展动作必须来自本地服务下发的结构化指令。
- 扩展按钮不能作为产品主流程的必要步骤。
- 页面动作必须使用白名单命令，例如 `collect_page`、`click`、`type`、`scroll`、`extract_table`，禁止执行 LLM 直接生成的任意 JavaScript。

## 10. 配置规范

`.env` 只保存开发机本地配置：

```text
OPENAI_API_KEY=
OPENAI_MODEL=
DESKPILOT_HOST=127.0.0.1
DESKPILOT_PORT=8765
DESKPILOT_DATA_DIR=data
```

用户设置保存 SQLite，不写入 `.env`。

## 11. 日志规范

日志分三类：

- 应用日志：后端启动、异常、API 调用。
- 任务日志：用户任务、计划、工具调用、结果摘要。
- 审计日志：敏感操作审批、拒绝、取消。

敏感内容处理：

- 微信聊天正文默认不进入通用日志。
- 截图默认只保存在任务目录。
- 上传到 OpenAI 前必须在任务记录中标记。

## 12. 测试规范

后端测试：

- API 使用 pytest + FastAPI TestClient。
- 工具 handler 需要单元测试。
- Agent workflow 至少要有 happy path 测试。

前端测试：

- 第一版至少保证构建通过。
- 关键交互后续补组件测试。

浏览器/RPA 测试：

- 网页自动化使用 Playwright trace。
- RPA 测试优先用固定截图样本，不依赖真实微信窗口。

## 13. 安全规范

默认必须确认：

- 读取聊天记录。
- 发送消息或提交表单。
- 删除、覆盖文件。
- 上传本地文件。
- 执行外部脚本。
- 安装软件。

默认可自动执行：

- 获取当前窗口标题和进程名。
- 总结用户当前打开的网页。
- 创建新的本地导出文件。
- 打开应用。
- 播放音乐。

## 14. 提交前检查

每次开发任务完成前至少执行：

```text
uv run pytest
npm run build
```

如果某项无法执行，必须在交付说明中写明原因。
