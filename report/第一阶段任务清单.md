# DeskPilot 第一阶段任务清单

## 1. 第一阶段目标

第一阶段只做项目骨架和“网页总结并保存”的受限 tool calling Agent 闭环，不做微信和网易云音乐。

浏览器扩展不应要求用户点击扩展按钮来触发网页总结。用户主入口固定为 DeskPilot 悬浮窗/应用发送按钮；扩展作为浏览器侧 bridge，接受后端指令采集当前页，并为后续网页自动操作复用同一条通信通道。

验收链路：

```text
打开网页
  -> 唤起 Tauri 悬浮窗
  -> 输入“总结当前网页并保存”
  -> FastAPI 创建任务
  -> LangGraph 识别 web_page_summary
  -> 进入受限 OpenAI tool calling loop
  -> LLM 调用 browser.collect_current_page
  -> 扩展采集当前页并回传 observation
  -> LLM 生成总结内容并调用 file.write_markdown
  -> SSE 推送完成
```

## 2. 任务拆分

### 2.1 后端骨架

- 创建 `backend/app/main.py`。
- 创建 FastAPI app。
- 增加 `/health`。
- 增加 `/chat`。
- 增加 `/events`。
- 增加 `/context/browser/current-page`。
- 增加浏览器扩展 WebSocket 通道，例如 `/browser/ws`。
- 增加 SQLite 初始化。
- 增加 Python logging。

### 2.2 Agent 骨架

- 创建 Agent state。
- 创建 intent router。
- 创建受限 OpenAI tool calling loop。
- 为 `web_page_summary` 定义工具白名单。
- 实现 ToolDefinition 到 OpenAI tools schema 的 adapter。
- 接入 OpenAI SDK。
- 接入 LangGraph checkpoint。
- 写入 task_runs 和 task_steps。

### 2.3 工具骨架

- 实现 Tool Registry。
- 为工具补充 `input_schema` 和 OpenAI function name 映射。
- 实现 `browser.collect_current_page`，通过扩展通道主动采集当前页。
- 实现 `browser.get_current_page`。
- 实现 `file.write_markdown`。
- 实现工具结果协议。

### 2.4 浏览器扩展

- 创建 Manifest V3 扩展。
- 实现 content script。
- 读取 URL、title、visible text。
- background service worker 建立本地 WebSocket 连接。
- 支持 `collect_page` 指令：后端请求时采集当前页并回传。
- 保留 `/context/browser/current-page` 作为开发期兼容/兜底，不作为用户主流程。

### 2.5 Tauri 前端

- 创建 Tauri + React + Vite + TypeScript 项目。
- 创建悬浮窗 UI。
- 创建设置页。
- 实现 `/chat` 调用。
- 实现 SSE 任务事件显示。

### 2.6 验证

- 后端 `uv run pytest` 通过。
- 前端 `npm run build` 通过。
- 浏览器扩展能连接本地服务，并响应 `collect_page` 指令。
- LLM 能通过 tool calling 自主调用 `browser.collect_current_page` 和 `file.write_markdown`。
- 网页总结能生成 Markdown 文件。

## 3. 不在第一阶段做

- 微信 RPA。
- 网易云音乐。
- OCR。
- PaddleOCR 集成。
- pyautogui 点击。
- 复杂网页操作业务闭环。
- 向量检索。
- 插件市场。

这些能力在第一阶段只保留目录和接口位置，不实现业务闭环。
