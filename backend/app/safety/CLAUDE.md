# safety — 安全与权限

所有工具调用前必须经过此模块的权限检查和风险评估。

## 风险等级

| 等级 | 含义 |
|---|---|
| `low` | 读取当前窗口标题、总结网页、创建本地文件 |
| `medium` | 读取结构化数据、导出文件、打开应用 |
| `high` | 读取聊天记录、发送消息、修改文件、删除操作 |

## 默认必须确认（approval required）

- 读取聊天记录、通讯录、浏览器历史
- 发送消息、发邮件、提交表单
- 删除文件、覆盖文件
- 修改系统设置
- 安装软件、执行外部脚本
- 上传本地文件到外部服务

## 默认可自动执行（auto allowed）

- 获取当前窗口标题和进程名
- 总结用户当前打开的网页
- 创建新的本地导出文件（不覆盖）
- 打开应用
- 播放音乐

## 审批流程

```
工具调用请求 → 查询权限策略（policy.py）
  → 评估风险等级（risk.py）
  → 需要确认? → 创建审批请求 → SSE 推送到前端
  → 等待用户确认/拒绝 → 记录审计日志（audit_log）
  → 放行/阻止工具执行
```

## 审计日志

敏感操作必须记录：
- 操作类型、目标、参数摘要
- 审批状态（approved / rejected / cancelled）
- 时间戳

微信聊天正文、截图 OCR 原文等敏感内容默认不进入通用日志。

## 文件组织

```
safety/
  policy.py      # 权限策略定义与查询
  risk.py        # 风险等级评估
  approvals.py   # 审批创建与状态管理
```

## 依赖

- 依赖：`db`（记录审批和审计日志）
- 不可依赖：`api`、`agent`、`tools`
